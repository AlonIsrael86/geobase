# GEOBase

GEOBase - פלטפורמת ניהול תוכן GEO עבור סוכנויות SEO

מערכת לאיסוף שאלות ותשובות מלקוחות ויצירת מאמרים מותאמים ל-Generative Engine Optimization.

## 🎯 מטרת הפלטפורמה

המערכת מאפשרת לצוות המכירות של 8200academy לאסוף שאלות ותשובות אמיתיות מלקוחות, לארגן אותן למאמרים, ולפרסם ישירות לוורדפרס בפורמט המותאם ל-AI/GEO.

## 📱 דפים

### 1. דאשבורד ניהול (`/dashboard`)

מרכז השליטה הראשי.

**כפתורי פעולה בכותרת (מותאם מובייל - נערמים מתחת ל-768px):**
| כפתור | סוג | פעולה |
|-------|-----|-------|
| הוסף שאלה + | Primary (ירוק) | מעבר לטופס הגשה |
| ייצוא לאקסל | Secondary (מסגרת) | יצוא כל השאלות לקובץ XLSX |
| ייבוא נתונים | Secondary (מסגרת) | ייבוא משאילת CSV / Excel / Google Sheets |
| צור מאמר + | Secondary (מסגרת) | מופעל רק כאשר נבחרו 2‑7 שאלות |
| מאגר ידע | Secondary (מנוטרל) | בקרוב – מחובר ל-AI |

**כרטיסי סטטיסטיקה (ברוחב מלא במובייל):**
| כרטיס | תיאור |
|-------|-------|
| ממתינות לטיפול | כמות השאלות החדשות שלא שובצו עדיין |
| בעריכה | כמות המאמרים בטיוטה |
| פורסמו | כמות המאמרים שפורסמו לוורדפרס |

**טבלת שאלות ותשובות (גלילה אופקית במובייל, צ'קבוקס מקובע):**
- עמודות (RTL): ☐ שאלה | קטגוריה (רוחב קבוע) | תאריך | סטטוס | פעולות
- פילטור לפי סטטוס וקטגוריה
- בחירה מרובה עם צ'קבוקסים
- כפתור עין (👁) לצפייה בשאלה המלאה
- כפתור מחיקה (🗑) לכל שורה

**סרגל פעולות צף (Floating Action Bar):** רוחב מלא במובייל, מופיע כשיש בחירה.
- RTL סדר כפתורים: נבחרו X שאלות | מחק נבחרות (אדום) | צור מאמר מהנבחרות | בטל בחירה
- מחיקה מרובה מציגה אישור ומוחקת מה-LocalStorage

**מודאל צפייה בשאלה:**
- קטגוריה ותאריך בראש
- תמונה (אם יש)
- שאלה מלאה
- תשובה מלאה (לא מקוצרת)
- מקור (אם יש)
- מספר מילים וסטטוס
- כפתורי "ערוך" ו"מחק"

**מודאל "כל השאלות והתשובות":**
- חיפוש + פילטר קטגוריה
- מסך מלא במובייל
- כפתור X יחיד בצד שמאל (RTL)

**מודאל מאגר ידע (UI בלבד, מנוטרל):**
- העלאת קבצי PDF/DOCX/TXT או הדבקת טקסט
- כפתור שמירה מנוטרל (בקרוב חיבור ל-AI)

### 2. טופס הגשה (`/submit`)

טופס מובייל-פירסט לצוות המכירות למילוי אחרי שיחות עם לקוחות.

**שדות:**
| שדה | סוג | הסבר |
|-----|-----|------|
| קטגוריה * | בחירה | אחת מ-6 קטגוריות |
| השאלה * | טקסט | מה הלקוח שאל? |
| התשובה * | טקסט ארוך | מומלץ 100-200 מילים |
| תמונה | קובץ | תמונה תוצג בראש התשובה |
| מקור | טקסט | לדוגמה: שיחה עם דני, 5.12.24 |

**מונה מילים:**
- 🟡 מתחת ל-100 מילים - אזהרה
- 🟢 100-200 מילים - תקין
- 🔴 מעל 200 מילים - ארוך מדי

**לאחר שליחה:**
- מסך הצלחה עם אנימציה
- אפשרות "הוסף שאלה נוספת" או "חזרה לדאשבורד"
- הנתונים נשמרים ב-localStorage

### 3. עורך מאמרים (`/articles/[id]`) - בקרוב

עריכת מאמר מלאה באפליקציה.

**תכונות מתוכננות:**
- כותרת ניתנת לעריכה
- בלוקים של שאלה-תשובה
- סיידבר עם גרירה לשינוי סדר
- כפתורי "שמור טיוטה" ו"פרסם לוורדפרס"

## 📊 מודל נתונים

```typescript
// שאלה ותשובה
interface Submission {
  id: string;
  question: string;        // השאלה
  answer: string;          // התשובה
  category: string;        // קטגוריה (אחת מ-6)
  imageUrl?: string;       // תמונה (אופציונלי)
  source?: string;         // מקור (אופציונלי)
  wordCount: number;       // מספר מילים
  status: "new" | "in_article" | "published";
  articleId?: string;      // מזהה מאמר (אם שובץ)
  createdAt: string;       // ISO date string
}

// מאמר
interface Article {
  id: string;
  title: string;           // כותרת
  status: "draft" | "editing" | "published";
  submissionIds: string[]; // שאלות במאמר
  wordpressUrl?: string;   // קישור לוורדפרס
  metaTitle?: string;      // כותרת SEO
  metaDesc?: string;       // תיאור SEO
  focusKeyword?: string;   // מילת מפתח
  publishedAt?: string;
  createdAt: string;
  updatedAt: string;
}
```

## 💾 אחסון נתונים

הנתונים נשמרים ב-localStorage של הדפדפן:

```typescript
// lib/storage.ts
getSubmissions()            // מחזיר את כל השאלות
addSubmission(data)         // מוסיף שאלה חדשה
addSubmissions([])          // ייבוא גורף של שאלות
getSubmissionById(id)       // מחזיר שאלה לפי ID
updateSubmission(id, data)  // מעדכן שאלה
deleteSubmission(id)        // מוחק שאלה
deleteSubmissions(ids[])    // מוחק רשימת שאלות

getArticles()              // מחזיר את כל המאמרים
addArticle(data)           // מוסיף מאמר חדש
getArticleById(id)         // מחזיר מאמר לפי ID
updateArticle(id, data)    // מעדכן מאמר
deleteArticle(id)          // מוחק מאמר

getCategories()            // מחזיר רשימת קטגוריות
addCategory(name)          // מוסיף קטגוריה
deleteCategory(name)       // מוחק קטגוריה
seedCategories()           // מאתחל קטגוריות ברירת מחדל

getStats()                 // מחזיר סטטיסטיקות לדאשבורד
seedInitialData()          // מאתחל נתוני דמו (פעם ראשונה)
```

| סטטוס | תווית | צבע | שימוש |
|-------|-------|-----|-------|
| new | חדש | 🔵 כחול | שאלות חדשות ממתינות לטיפול |

### RTL (ימין לשמאל)

כל הממשק מותאם ל-RTL:
- לוגו וכותרת בצד **ימין**
- כפתורי פעולה בצד **שמאל**
- אייקונים מופיעים **אחרי** הטקסט (לדוגמה: "הוסף שאלה +")
- מספרים מוצגים ב-LTR (תאריכים, מילים)
- כפתור X לסגירת מודאל בצד **שמאל**
- עמודת צ'קבוקס **ראשונה** (בצד ימין)

### פונט
- **Heebo** - פונט עברי מודרני מ-Google Fonts
- משקלים: 300-900
